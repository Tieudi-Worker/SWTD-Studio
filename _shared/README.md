# _shared/

Shared assets referenced by all 13 HMA skills. Markdown for human-readable doctrine and prompts; JSON for machine-readable rules and configs.

## Layout

```
_shared/
├── doctrine/        # Constitution (always-true rules)
├── rules/           # Anti-patterns + per-slot constraints
├── prompts/         # Slot prompt templates
├── config/          # Image specs, model routing, funnels, research
├── knowledge/       # Cross-SKU knowledge base
└── schema/          # JSON Schemas (brief, etc.)
```

## Files

### doctrine/
- `handmade-product-doctrine.md` — F1-F8 fidelity rules, image strategy, conversion doctrine. **Read first**.
- `seedance-ugc-video-doctrine.md` — Video-specific doctrine.

### rules/
- `prompt-anti-patterns.md` — A1-A8 universal patterns + RULE-015 → RULE-021 per-slot rules.
- `slot-constraints.json` — Machine-readable: per-slot `max_words`, `hard_max`, `forbidden_phrases`, `inject` flags. Consumed by `runtime/lib/prompt-guard.mjs`.

### prompts/
- `slot-prompts.json` — Fallback prompt templates per slot (1-8) and per A+ module (M1-M5).

### config/
- `image-specs.json` — Output dimensions: listing 2000×2000 q95, A+ 1464×600 q95, video 1920×1080 20-25s.
- `model-routing.json` — Per-slot model selection (nano-banana-pro primary, seedream-4 fallback).
- `research-config.json` — Auto-research keyword rules.
- `funnels/`
  - `gift_ornament.json` — 8-slot funnel for wood/acrylic/frame ornaments.
  - `daily_use.json` — Funnel for daily-use products.
  - `paper_card.json` — Paper card funnel.
  - `aplus-type1-single.json` — A+ Plan B Storyboard seamless (Type 1 single ASIN).
  - `aplus-type2-multi-asin.json` — A+ Plan A Modular comparison (Type 2 multi-ASIN).
  - `aplus-shared-constraints.json` — R1-R4 cross-Type rules (typography, fidelity, scale, ref selection).
  - `_default.json` — Fallback funnel.

### knowledge/
- `_index.json`, `_summary.json` — Auto-generated index + summary, regenerated by `/hma-knowledge`.
- `amazon-listing/` — `conversion-tactics.md`, `image-requirements.md`.
- `photography/` — `product-photography.md`, `product-structure-analysis.md` (mandatory pre-read for multi-layer SKUs).
- `niches/` — Per-niche learnings (`acrylic-ornaments.md`, `reward-jars.md`).
- `brand-style/` — Per-shop branding: `_template.json`, `<brand>.json` (e.g. `sweetdtd.json`).

### schema/
- `brief.schema.json` — JSON Schema for `data/<SKU>/brief.json`, validated by `runtime/lib/brief-validator.mjs`.

## Editing Discipline

- Markdown files (doctrine, rules, knowledge) are the source of truth. Edit them, then re-run `/hma-knowledge index` to refresh `_index.json` and `_summary.json`.
- JSON files (slot-constraints, image-specs, model-routing, funnels) are consumed by runtime code. Schema-validate after editing.
- Do **not** copy edits back to `D:\AI PROJECT\handmade-media-agent-v0\` unless the user explicitly asks for back-port. That folder is frozen.
